# Post para LinkedIn - StickerLab

**Álbum de figurinhas + código + COPA DO MUNDO ⚽🏆 = StickerLab**

A Copa do mundo de futebol 2026 chegou e uma curiosidade sempre me intrigou: **qual a taxa de acerto da Panini nas convocações prévias do álbum?**

Pesquisei, mas dificilmente se encontra uma API disponível com essas informações completas e estruturadas. A FIFA disponibiliza a lista oficial após as convocações, as imagens do álbum existem, então resolvi criar minha própria solução.

Por que não transformar essa curiosidade em código?

Juntando o meu entusiasmo e minha vontade de sempre estar aprendendo algo novo, desenvolvi o **StickerLab** - um app que cruza os dados do álbum com as convocações oficiais:

- Qual a taxa de acerto geral da Panini na elaboração prévia do álbum?
- Quantos jogadores do álbum foram convocados?
- Quantos foram convocados mas não têm figurinha?
- Qual seleção tem a melhor taxa de acerto?
- Qual o clube (time) mais foi representado na copa ?

Essas perguntas refletem o mundo real do desenvolvimento: **identificar problemas, estruturar questões relevantes e entregar insights que agregam valor ao usuário final**. No dia a dia, é assim que resolvemos problemas reais - entendendo o que importa para quem vai usar a solução.

**Stack:**

- Backend: Node.js + PostgreSQL + Docker + AWS
- Frontend: Next.js 16 + React 19 + TypeScript + Tailwind

**Como foram extraídos os dados:**

- Lista oficial das convocações: extraída diretamente da FIFA (fonte oficial)
- Lista do álbum: extraída automaticamente das imagens reais do álbum Panini, tratadas e estruturadas em JSON
- Cruzamento e análise: algoritmo desenvolvido do zero para comparar os dados, calcular taxas de acerto por seleção, identificar discrepâncias e gerar estatísticas em tempo real

**O diferencial:** desenvolvi todo o projeto utilizando ferramentas de IA como auxílio e aceleração.

Desde a extração e estruturação dos dados, comparação das informações, criação das análises estatísticas, modelagem do banco de dados, desenvolvimento do backend e frontend, até a documentação completa - cada etapa foi desenvolvida por mim com o suporte de agents para otimizar o processo.

**O mais impressionante?** Não foi só a velocidade (um projeto que levaria semanas ficou pronto em dias), mas a qualidade: informações precisas, código consistente e estável. Essa abordagem acelerou cada etapa mantendo qualidade técnica e concisão. Foi minha forma de aprender novas tecnologias enquanto testava o quão eficiente e robusto pode ser o desenvolvimento moderno.

**Reflexão:** Este projeto me fez perceber algo importante - não estamos apenas surfando uma onda nova. O futuro já chegou, e precisamos nos reinventar a cada dia.
